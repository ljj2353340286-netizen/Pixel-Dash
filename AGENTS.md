# AGENTS.md

## Project Overview
Pixel Dash is a 2D endless runner game developed for the Software Engineering course.  
The purpose of this project is to build a simple playable game while applying software engineering practices such as Agile development, documentation, testing, and repository-based development.

## Environment
- HTML
- CSS
- JavaScript
- GitHub

## Project Structure
- `/assets`: game images and resources
- `/docs`: project documentation
- `/docs/adr`: architecture decision records
- `/docs/user-story`: user stories
- `/docs/ui`: UI design documents
- `/docs/sprint`: sprint plans and reports
- `/skills`: reusable development notes
- `/test`: test case documents
- `/todo`: sprint task lists

## Code Conventions
- Use clear variable and function names.
- Use camelCase for JavaScript variables and functions.
- Keep the game logic simple and readable.
- Add comments for important logic such as jumping, collision detection, and scoring.

## Constraints
- The game should run in a web browser.
- The project should remain simple and maintainable.
- All important development decisions should be documented.
- Testing documents should be updated when major features are changed.
